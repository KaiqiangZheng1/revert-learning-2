🧩 Question 2 – Average memory usage (5-minute intervals) Goal: Calculate average used memory % per host for every 5-minute window. (Used memory = total memory − free memory) Step 1️⃣ Create a helper function to round timestamps CREATE OR REPLACE FUNCTION round5(ts TIMESTAMP) RETURNS TIMESTAMP AS $$ BEGIN RETURN date_trunc('hour', ts) + (date_part('minute', ts)::int / 5) * INTERVAL '5 min'; END; $$ LANGUAGE plpgsql; Step 2️⃣ Insert some sample data for testing (if needed) INSERT INTO host_usage ("timestamp", host_id, memory_free, cpu_idle, cpu_kernel, disk_io, disk_available) VALUES ('2019-05-29 15:00:00.000', 1, 300000, 90, 4, 2, 3), ('2019-05-29 15:01:00.000', 1, 200000, 90, 4, 2, 3), ('2019-05-29 15:02:00.000', 1, 100000, 90, 4, 2, 3), ('2019-05-29 15:06:00.000', 1, 150000, 90, 4, 2, 3); Step 3️⃣ Query the average memory usage percentage SELECT hu.host_id, hi.hostname, round5(hu."timestamp") AS ts_5min, ROUND(AVG((hi.total_mem - hu.memory_free) * 100.0 / hi.total_mem)) AS avg_used_mem_percentage FROM host_usage hu JOIN host_info hi ON hu.host_id = hi.id GROUP BY hu.host_id, hi.hostname, round5(hu."timestamp") ORDER BY hu.host_id, ts_5min; ✅ Explanation: round5(timestamp) → groups timestamps into 5-minute windows (hi.total_mem - hu.memory_free) → used memory AVG(... * 100 / total_mem) → average used memory percentage GROUP BY both host_id and rounded timestamp 🧩 Question 3 – Detect host failure Goal: A host fails if it logs fewer than 3 entries in any 5-minute window. SQL Query SELECT hu.host_id, hi.hostname, round5(hu."timestamp") AS ts_5min, COUNT(*) AS num_data_points FROM host_usage hu JOIN host_info hi ON hu.host_id = hi.id GROUP BY hu.host_id, hi.hostname, round5(hu."timestamp") HAVING COUNT(*) < 3 ORDER BY hu.host_id, ts_5min; ✅ Explanation: Groups by each host and 5-minute time slot Uses HAVING COUNT(*) < 3 to detect missing data points (possible failure) Shows you which host and time window failed
